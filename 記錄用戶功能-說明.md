# 📝 記錄用戶功能 - 使用說明

## 🎯 功能簡介

在檢查用戶是否可以發送訊息時，現在可以將用戶的 Instagram 鏈接記錄到文本文件中，方便後續查看和管理。

## ✨ 新增按鈕

在用戶審核對話框中，新增了一個**「📝 記錄下來」**按鈕：

```
┌────────────────────────────┐
│      用戶 3/12578          │
│   @amanoyuki5              │
│                            │
│ 📌 請在瀏覽器中檢查該用戶頁面 │
│ 查看是否有【訊息】按鈕      │
│                            │
│  [✅ 可以發送] [⏭️ 跳過] [🛑 停止] │
│                            │
│      [📝 記錄下來]          │
└────────────────────────────┘
```

## 🚀 使用方法

1. **開始群發任務**
   - 匯入 IG 帳號清單
   - 點擊「開始群發」

2. **檢查用戶**
   - 系統會逐個顯示用戶資訊
   - 在瀏覽器中檢查該用戶頁面

3. **記錄用戶**
   - 如果想記錄當前用戶的鏈接
   - 點擊「📝 記錄下來」按鈕
   - 用戶鏈接會自動保存到文件中

4. **繼續處理**
   - 記錄完成後
   - 可以選擇「✅ 可以發送」、「⏭️ 跳過」或「🛑 停止」

## 📄 記錄文件格式

所有記錄會保存到項目根目錄的 `recorded-users.txt` 文件中。

**文件格式**：每行一個完整的Instagram鏈接

**範例**：
```
https://www.instagram.com/amanoyuki5/
https://www.instagram.com/john_doe/
https://www.instagram.com/jane_smith/
```

**特點**：
- ✅ 簡潔明了，只包含鏈接
- ✅ 每行一個鏈接
- ✅ 可以直接用於批量操作
- ✅ 方便複製粘貼

## 💡 使用場景

1. **篩選目標用戶**
   - 記錄有興趣的用戶
   - 建立專屬的用戶清單

2. **追蹤重要帳號**
   - 記錄需要特別關注的帳號
   - 方便後續追蹤

3. **建立備份清單**
   - 保存已檢查過的用戶
   - 作為日後參考

4. **多次發送準備**
   - 記錄可以發送訊息的用戶
   - 用於下次批量發送

## 🎨 按鈕狀態

- **正常狀態**：📝 記錄下來（綠色）
- **處理中**：⏳ 記錄中...（灰色，不可點擊）
- **成功**：✅ 已記錄（綠色，1.5秒後恢復）
- **失敗**：❌ 記錄失敗（紅色，1.5秒後恢復）

## 📂 文件位置

記錄文件位於：
```
C:\Projects\ig-dm-helper\recorded-users.txt
```

## 🔄 重複記錄

- 可以重複點擊「記錄下來」按鈕
- 每次點擊都會在文件末尾追加一行新記錄
- 包含最新的時間戳記

## 💾 查看記錄

使用任何文本編輯器打開 `recorded-users.txt` 即可查看所有記錄。

建議使用：
- 記事本
- Notepad++
- VS Code
- 或任何文本編輯器

## ⚠️ 注意事項

1. 記錄文件會持續增長，建議定期清理舊記錄
2. 每條記錄都包含時間戳記，方便追蹤記錄時間
3. 文件使用 UTF-8 編碼，支援中文和特殊字符
4. 記錄操作不會影響群發流程，可以隨時記錄

## 🎉 開始使用

現在就試試這個新功能吧！在檢查用戶時，隨時點擊「📝 記錄下來」按鈕，輕鬆管理你的用戶清單！
