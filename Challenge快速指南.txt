🎯 快速處理（3 分鐘）
-----------------------------------------

### 步驟 1：點【🌐 打開 Instagram】
- 自動在瀏覽器打開 https://www.instagram.com

### 步驟 2：登入並完成驗證
- 輸入帳號密碼
- 完成 Instagram 要求的驗證（見下方詳情）

### 步驟 3：返回程序點【🔄 重新嘗試】
- 自動清理舊數據
- 彈出登入對話框
- 重新登入

### 步驟 4：繼續使用
- ✅ 完成！可以群發了


📝 可能的驗證類型
-----------------------------------------

### 類型 A：輸入驗證碼 ⏱️ 2-3 分鐘

```
Instagram 顯示：
「為了確保是你本人，請輸入驗證碼」

要做的事：
1. 選擇接收方式（郵箱/手機）
2. 查看驗證碼（檢查郵箱或簡訊）
3. 輸入驗證碼到 Instagram
4. 點擊【確認】
5. ✅ 完成
```

### 類型 B：確認登入位置 ⏱️ 1-2 分鐘

```
Instagram 顯示：
「我們注意到從新位置登入」
「這是你嗎？」

要做的事：
1. 確認是你本人操作
2. 點擊【是我本人】或【This Was Me】
3. ✅ 完成
```

### 類型 C：回答安全問題 ⏱️ 3-5 分鐘

```
Instagram 顯示：
「請回答你的安全問題」

要做的事：
1. 閱讀安全問題
2. 輸入正確答案
3. 點擊【提交】
4. ✅ 完成
```

### 類型 D：驗證手機號碼 ⏱️ 3-5 分鐘

```
Instagram 顯示：
「請驗證你的手機號碼」

要做的事：
1. 輸入手機號碼
2. 接收驗證碼（簡訊）
3. 輸入驗證碼
4. 點擊【確認】
5. ✅ 完成
```

💡 小技巧
-----------------------------------------

### 技巧 1：保持瀏覽器開著

- ✅ 完成驗證後不要關閉瀏覽器
- ✅ 可能需要再次驗證時會更快

### 技巧 2：使用同一台電腦

- ✅ 在運行本程序的電腦上完成驗證
- ✅ 不要在手機或其他設備上驗證

### 技巧 3：準備好驗證方式

- ✅ 確保手機在身邊（接收簡訊）
- ✅ 確保能訪問郵箱（接收驗證碼）

### 技巧 4：降低群發頻率

- ✅ 完成驗證後，降低發送頻率
- ✅ 避免短時間內再次觸發


❌ 常見錯誤
-----------------------------------------

### 錯誤 1：在手機上完成驗證

❌ 問題：
- 在手機上登入並完成驗證
- 返回程序後還是失敗

✅ 解決：
- 必須在運行本程序的電腦上完成驗證
- Instagram 會檢查設備和 IP

### 錯誤 2：驗證後立即關閉瀏覽器

❌ 問題：
- 完成驗證後立即關閉瀏覽器
- 可能導致驗證失效

✅ 解決：
- 保持瀏覽器開著
- 至少等待 5 分鐘

### 錯誤 3：沒有點【重新嘗試】

❌ 問題：
- 完成驗證後直接點【關閉】
- 然後直接點【群發】
- 還是出現錯誤

✅ 解決：
- 必須點【🔄 重新嘗試】
- 清理舊數據並重新登入

### 錯誤 4：使用 VPN

❌ 問題：
- 使用 VPN 完成驗證
- Instagram 檢測到 IP 改變
- 可能再次觸發驗證

✅ 解決：
- 關閉 VPN
- 使用穩定的網絡


🔄 如果還是失敗
-----------------------------------------

### 方案 A：等待更長時間

```
1. 完成驗證後等待 15-30 分鐘
2. 確保在瀏覽器中已經成功登入
3. 再次在本程序中嘗試
```

### 方案 B：重啟程序

```
1. 完全關閉本程序
2. 關閉瀏覽器
3. 等待 5 分鐘
4. 運行 .\quick-fix.bat
5. 重新登入
```

### 方案 C：更換帳號

```
1. 如果某個帳號頻繁出現 Challenge
2. 建議暫時停用該帳號
3. 使用其他帳號
4. 等待 24 小時後再試
```


📞 需要幫助？
-----------------------------------------

### 檢查終端輸出

查看詳細的錯誤信息：
```
⚠️ 重定向次數過多！可能原因：
   1. 帳號需要驗證（Challenge）  ← 這個
   2. Session 已過期
   3. 帳號被臨時限制
   👉 建議：在瀏覽器登入 Instagram 完成驗證後再試
```

### 確認驗證完成

在瀏覽器中檢查：
- ✅ 能看到 Instagram 首頁
- ✅ 能看到自己的個人資料
- ✅ 能發送訊息
- ✅ 沒有任何警告或限制提示
