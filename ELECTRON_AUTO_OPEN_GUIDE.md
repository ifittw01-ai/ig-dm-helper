# 🚀 Electron 應用自動打開 IG 用戶頁面功能

## ✨ 功能說明

現在，當您在 Electron 應用中點擊**"開始群發"**按鈕時，程序會：

1. ✅ **自動在瀏覽器中打開所有目標用戶的 Instagram 主頁**
2. ✅ **每個用戶間隔 2 秒打開**
3. ✅ **顯示進度狀態**
4. ✅ **打開完成後詢問是否繼續發送**

---

## 🎮 使用流程

### 步驟 1：啟動應用

```bash
npm start
```

### 步驟 2：登入 Instagram

在應用中輸入您的 Instagram 用戶名和密碼。

### 步驟 3：匯入用戶清單

點擊"匯入 IG 帳號"按鈕，選擇包含用戶名的 TXT 文件。

### 步驟 4：輸入消息

在消息框中輸入要發送的內容。

### 步驟 5：點擊"開始群發"

點擊按鈕後會出現確認對話框：

```
確定要向 10 個帳號發送訊息嗎？

步驟：
1. 程序會先在瀏覽器中打開所有用戶頁面
2. 請在瀏覽器中檢查每個用戶是否有【訊息】按鈕
3. 然後開始發送訊息

注意：頻繁操作可能導致帳號被限制。

[確定] [取消]
```

### 步驟 6：自動打開用戶頁面

點擊確定後：

```
狀態：🌐 正在打開用戶頁面
準備打開 10 個用戶頁面...

[1/10] 已打開 @user1
(等待 2 秒...)

[2/10] 已打開 @user2
(等待 2 秒...)

...

[10/10] 已打開 @user10
```

瀏覽器會自動打開每個用戶的 Instagram 主頁！

### 步驟 7：檢查用戶主頁

在瀏覽器的各個標籤頁中：

- ✅ 查看用戶是否有【訊息】按鈕
- ✅ 記下可以發送消息的用戶
- ❌ 沒有【訊息】按鈕的用戶不能發送

### 步驟 8：確認是否繼續

打開完成後會顯示：

```
✅ 已在瀏覽器中打開所有 10 個用戶頁面！

請在瀏覽器中查看：
- 每個用戶是否有【訊息】按鈕
- 記下可以發送消息的用戶

確認後是否開始發送訊息？

[確定] [取消]
```

### 步驟 9：開始發送

點擊確定後，程序開始批量發送消息。

## ⚙️ 配置選項

### 修改打開間隔

在 `renderer.js` 第 263 行：

```javascript
await openAllProfiles(usernames, 2000); // 2000 = 2 秒
```

可以修改為：
- `1000` = 1 秒（快速）
- `2000` = 2 秒（推薦）
- `3000` = 3 秒（較慢）
- `5000` = 5 秒（仔細查看）

## 💡 使用技巧

### 技巧 1：分批處理

如果用戶太多（>20 個）：
- 分成多批匯入
- 每批 10-15 個
- 避免瀏覽器標籤頁太多

### 技巧 2：快速檢查

打開所有用戶頁面後：
- 快速瀏覽每個標籤頁
- 查看是否有【訊息】按鈕
- 記下不能發送的用戶

### 技巧 3：優化用戶清單

在匯入前先篩選：
- 移除不活躍的用戶
- 只保留公開帳號
- 提高成功率

## 📋 檢查清單

### 發送前檢查

在瀏覽器中打開用戶頁面後：

- [ ] 用戶是否存在（沒有顯示"用戶不存在"）
- [ ] 是否有【訊息】按鈕
- [ ] 用戶是否活躍（有最近帖子）
- [ ] 粉絲數是否符合目標
- [ ] 帳號類型是否正確（公開/私密）

### 發送時注意

- [ ] 消息內容已檢查
- [ ] 用戶清單已確認
- [ ] 已準備好暫停/停止按鈕
- [ ] 不要過於頻繁發送

## ⚠️ 注意事項

### 1. 瀏覽器標籤頁

- 打開太多標籤頁可能導致瀏覽器變慢
- 建議每次不超過 20 個用戶
- 可以分批處理

### 2. Instagram 限制

- 不要短時間內打開太多用戶頁面
- 避免被 Instagram 識別為異常行為
- 建議間隔 2-3 秒

### 3. 網絡連接

- 確保網絡連接穩定
- 打開頁面需要時間加載
- 等待頁面完全加載後再檢查

## 🔧 故障排除

### 問題 1：瀏覽器沒有打開

**原因：** 沒有設置默認瀏覽器

**解決：**
1. 設置默認瀏覽器
2. 重啟應用

### 問題 2：打開速度太快

**原因：** 間隔時間太短

**解決：**
1. 修改 `renderer.js` 中的延遲時間
2. 改為 3000 或 5000（毫秒）

### 問題 3：應用卡住

**原因：** 用戶太多，瀏覽器壓力大

**解決：**
1. 關閉一些標籤頁
2. 重啟應用
3. 減少每批用戶數量
