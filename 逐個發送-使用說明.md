# 逐個發送功能 - 使用說明

## ✨ 新功能特點

現在群發功能已升級為**逐個處理模式**，更加智能和安全！

### 🎯 工作流程

1. **打開用戶頁面** 📱
   - 程序會在瀏覽器中打開當前用戶的 Instagram 頁面
   - 等待 2 秒讓頁面完全加載

2. **彈出確認對話框** 💬
   - 在應用程序中會彈出一個漂亮的對話框
   - 顯示當前用戶名和進度（例如：用戶 1/10）

3. **你的選擇** 🤔
   - ✅ **可以發送**：確認該用戶有【訊息】按鈕，程序會立即發送訊息
   - ⏭️ **跳過**：該用戶不能接收訊息，跳過並處理下一個
   - 🛑 **停止**：停止整個批量發送任務

4. **自動處理下一個** 🔄
   - 處理完當前用戶後，自動打開下一個用戶頁面
   - 重複以上流程

## 📋 使用步驟

### 1️⃣ 準備工作
```
1. 確保已經登入 Instagram
2. 匯入要發送訊息的帳號清單
3. 輸入訊息文案
```

### 2️⃣ 開始發送
```
1. 點擊【開始群發】按鈕
2. 確認開始
3. 程序會自動打開第一個用戶的 IG 頁面
```

### 3️⃣ 檢查並確認
```
在瀏覽器中檢查：
- 是否有【訊息】或【Message】按鈕？
  ✅ 有 → 點擊應用中的【可以發送】
  ❌ 沒有 → 點擊應用中的【跳過】
```

### 4️⃣ 查看結果
```
完成後會顯示統計：
- ✅ 成功：X 個
- ⏭️ 跳過：X 個
- ❌ 失敗：X 個
```

## 💡 使用建議

### 關於瀏覽器頁面
- **建議**：在瀏覽器中保持同一個標籤頁
- 每次打開新用戶時，會在該標籤頁中刷新
- 這樣可以更快速地檢查每個用戶

### 關於發送速度
- 程序會在每個用戶之間自動延遲 1 秒
- 這是為了避免觸發 Instagram 的速率限制
- 建議不要一次性發送太多（建議 < 20 個）

### 關於錯誤處理
如果發送失敗：
- 程序會自動記錄失敗原因
- 可以在任務列表中查看狀態
- 失敗的用戶可以稍後重試

## 🔧 常見問題

### Q: 瀏覽器打開的頁面太多怎麼辦？
A: 新版本不會批量打開所有頁面，只會一個一個打開，所以不用擔心！

### Q: 如果我不確定是否可以發送怎麼辦？
A: 選擇【跳過】，稍後可以手動處理該用戶。

### Q: 可以中途停止嗎？
A: 可以！在確認對話框中選擇【停止】，或點擊主界面的停止按鈕。

### Q: 發送失敗會怎樣？
A: 程序會自動記錄失敗，並繼續處理下一個用戶。

### Q: 瀏覽器頁面不會自動刷新？
A: 這是正常的。`shell.openExternal()` 會在瀏覽器中打開新標籤。建議手動切換到該標籤或使用書籤管理。

## 📊 進度顯示

在主界面可以看到：
- 當前狀態：打開頁面 / 發送中 / 完成
- 進度條：視覺化顯示完成百分比
- 任務計數：1 / 10
- 每個用戶的狀態：成功 ✅ / 跳過 ⏭️ / 失敗 ❌
