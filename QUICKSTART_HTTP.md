# ⚡ 快速開始 - HTTP API 版本

## 🎯 5 分鐘上手

### 第一步：安裝

```bash
# 進入專案目錄
cd ig-dm-helper

# 安裝依賴（如果還沒安裝）
npm install
```

### 第二步：啟動

```bash
npm start
```

### 第三步：登入

1. 應用啟動後，會看到登入畫面
2. 輸入您的 Instagram **帳號**
3. 輸入您的 Instagram **密碼**
4. 點擊「🚀 登入」按鈕

> ⚠️ **注意：** 目前不支援雙重驗證（2FA）的帳號

### 第四步：開始使用

登入成功後，您可以：

#### 📤 批量發送私訊

1. 點擊「📥 匯入 IG 帳號」
2. 選擇一個 `.txt` 文件（每行一個用戶名）
3. 在訊息框中輸入要發送的內容
4. 點擊「▶️ 開始群發」

#### 👥 匯出粉絲列表

1. 點擊「👥 匯出粉絲」
2. 輸入要匯出的 IG 帳號名稱
3. 等待抓取完成
4. 選擇保存位置

#### 💾 保存登入狀態

點擊「💾 保存登入狀態」按鈕，下次啟動可直接使用「初始化」功能。

---

## 📝 帳號清單格式

創建一個 `accounts.txt` 文件：

```
user1
user2
user3
user4
```

每行一個 Instagram 用戶名，不需要 `@` 符號。

## 💡 使用技巧

### 1. 避免被封鎖

- 每天發送數量控制在 **50 條以內**
- 使用 **隨機延遲**（已內建 30-60 秒）
- 不要發送 **重複內容**
- **個性化** 訊息內容

### 2. 首次登入建議

- 使用 **測試帳號** 先試用
- 確保帳號 **沒有開啟雙重驗證**
- 登入成功後立即 **保存登入狀態**

### 3. Session 管理

- **首次登入**：輸入帳號密碼，點擊「登入」
- **再次使用**：點擊「初始化」即可載入保存的狀態
- **切換帳號**：重新輸入帳號密碼登入

## 🐛 遇到問題？

### 登入失敗

**可能原因：**
- 帳號或密碼錯誤
- 帳號開啟了雙重驗證
- Instagram 偵測到異常

**解決方法：**
1. 確認帳號密碼正確
2. 關閉雙重驗證
3. 換一個網路環境試試
4. 使用測試帳號

### 發送失敗

**可能原因：**
- 目標用戶不存在
- 發送過於頻繁
- Session 過期

**解決方法：**
1. 確認用戶名正確（不含 @）
2. 減慢發送速度
3. 重新登入

### 無法啟動

**可能原因：**
- 依賴未安裝
- 端口被占用
- Node.js 版本過低

**解決方法：**
```bash
# 重新安裝依賴
rm -rf node_modules
npm install

# 確認 Node.js 版本 >= 16
node -v
```

function randomDelay() {
