===============================================
   🎉 新功能：遇到登入問題自動換帳號繼續發送
===============================================

✨ 新功能特點
--------------
❌ 以前：發送失敗 → 程序停止 → 要重新開始
✅ 現在：發送失敗 → 彈出對話框 → 換帳號登入 → 繼續發送！


🚀 立即測試（3 分鐘完成）
--------------------------

步驟 1：清理並啟動
------------------
1. 雙擊運行 quick-fix.bat
2. 等待程序啟動
3. 點擊【初始化】
4. 點擊【登入】（用帳號A登入）


步驟 2：開始群發
---------------
1. 匯入 3-5 個用戶
2. 輸入測試訊息
3. 點擊【開始群發】


步驟 3：模擬登入問題（可選）
---------------------------
如果想測試換帳號功能，你可以：

方法1：等待自然觸發
- 繼續發送
- 如果遇到 Session 過期或需要驗證
- 會自動彈出對話框

方法2：強制觸發（測試用）
- 發送幾個用戶後
- 在終端按 Ctrl+C 中斷（測試環境）
- 或者等待 Instagram 限制


步驟 4：換帳號繼續
-----------------
當彈出對話框時：

┌────────────────────────────────────┐
│  ⚠️ 檢測到登入問題                 │
│                                    │
│  錯誤：Session 已過期或需要驗證     │
│  當前用戶：@user3                  │
│                                    │
│  [🔄 重新登入並繼續] [🛑 停止]     │
└────────────────────────────────────┘

1. 點擊【重新登入並繼續】
2. 在登入框輸入帳號B的用戶名和密碼
3. 點擊【登入】按鈕
4. 等待登入成功
5. 點擊對話框中的【確認完成】

✅ 完成！程序會自動繼續發送剩餘用戶！


📋 預期結果
-----------
發送流程：
  user1 → ✅ 成功（用帳號A）
  user2 → ✅ 成功（用帳號A）
  user3 → ❌ 登入錯誤（帳號A失效）
  
  [彈出對話框]
  [你登入帳號B]
  
  user3 → ⏭️ 跳過（之前失敗的）
  user4 → ✅ 成功（用帳號B）← 換帳號後繼續！
  user5 → ✅ 成功（用帳號B）

最終統計：
  ✅ 成功：4 個
  ⏭️ 跳過：1 個
  ❌ 失敗：0 個


💡 實際使用場景
---------------

場景1：帳號被限制
  發送 20 個後被限制
  → 換帳號B繼續發送剩餘 30 個
  → 完成 50 個用戶的群發！

場景2：Session 過期
  發送中途 Session 過期
  → 重新登入（可以用同一帳號或換帳號）
  → 繼續發送！

場景3：需要驗證
  Instagram 要求驗證
  → 換一個不需要驗證的帳號
  → 繼續發送！


🎯 關鍵優勢
-----------
✅ 無縫切換 - 不會中斷流程
✅ 自動跳過 - 失敗的用戶自動跳過
✅ 靈活應對 - 隨時換帳號
✅ 節省時間 - 不用重新開始


⚠️ 重要提示
-----------
1. 跳過失敗用戶
   重新登入後，當前失敗的用戶會被跳過
   （避免重複發送或再次觸發限制）

2. 準備多個帳號
   建議準備 2-3 個帳號備用
   - 帳號A：主帳號
   - 帳號B：備用帳號1
   - 帳號C：備用帳號2

3. 快速切換
   提前記住備用帳號密碼
   遇到問題能快速切換

💬 常見問題
-----------

Q：換帳號後從哪裡繼續？
A：從下一個用戶開始，當前失敗的用戶會被跳過

Q：可以換不同的帳號嗎？
A：可以！可以用任何有效的 Instagram 帳號

Q：如果不想換帳號怎麼辦？
A：點擊【停止群發】按鈕，程序會停止並顯示統計

Q：重新登入需要多久？
A：通常 < 1 分鐘（輸入帳號密碼 + 登入驗證）

Q：會重複發送失敗的用戶嗎？
A：不會，失敗的用戶會被跳過，避免重複
